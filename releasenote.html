네이버 뮤직 for Mac 1.5.2

수정 사항

  1. 업데이트 이후 오른쪽 메뉴에서 “개발자 블로그” 메뉴가 작동하지 않던 것을 수정

기타

[개발자 전용] 미디어키를 제외하고 URI Scheme로 플레이어를 컨트롤 할 수 있는 기능이 추가되었습니다. 해당 URI Scheme는 맥의 전역에서 감지되고 작동합니다. 필요에 따라 애플스크립트나, bash 스크립트, 키 바인딩, 또는 매크로에 적용하셔도 좋습니다.

  • [재생/일시정지] navermusic://play_pause
  • [다음곡] navermusic://next
  • [이전곡] navermusic://prev

//브라우저에서 A 태그 사용시

<a href="navermusic://play_pause">재생/일시정지</a>
<a href="navermusic://next">다음곡</a>
<a href="navermusic://prev">이전곡</a>

또는 주소창에 상황에 맞게 다음 중 하나를 입력

navermusic://play_pause
navermusic://next
navermusic://prev

터미널 사용시

$ open navermusic://play_pause
$ open navermusic://next
$ open navermusic://prev

네이버 뮤직 for Mac 1.5.1

수정 사항

  1. Safari와 별도로 세션 분리적용
  • Safari와 로그인 세션을 별도로 적용합니다. 따라서, Safari의 로그인/로그아웃에 의해 플레이어의 로그인이 영향을 받지 않습니다.

기타

  1. Mac OS X 10.10 요세미티(Yosemite)에서 정상적으로 동작 확인하였습니다.
  2. 자동로그인의 경우 ‘네이버 로그인 세션 정책(쿠키 유효기간)’으로 인하여 2주에 한번정도 로그아웃이 될 가능성이 있습니다. 앱 내부에서 여러분의 소중한 아이디/패스워드를 다루지 않도록 하기 위함이니 양해해 주시길 바래요 :)

네이버 뮤직 for Mac 1.5.0

자세한 내용과 코멘트는 아래 링크를 참조 해주세요.

http://lunic.tistory.com/entry/네이버뮤직-for-Mac-150-업데이트-안내

기능 추가

  1. 자동로그인 추가 (Safari 세션 공유)
  2. 환경설정 추가
  3. 플레이어 새로고침 메뉴추가
  4. 라디오 모드 지원 시작
  5. 재생중일때 상태바 아이콘 색상이 변경되지 않는 기능이 추가
  6. 알림센터 Skip 버튼 지원
  7. Safari, Chrome 플러그인이 업데이트 되었습니다.

버그 수정사항

  1. 로그인 실패시 창 크기가 줄어들었던 현상 수정
  2. 내 리스트에 담기, MP3, 앨범버튼, 가수이름을 클릭시 아무런 반응이 없던 현상 수정
  3. 알림센터에 같은 곡의 알림이 여러번 누적되던것을 수정
  4. 플레이어에 왼쪽 상단에 있는 네이버 뮤직 로고 클릭시 아무런 반응이 없던 현상 수정
  5. 같은 ID로 다른 기기에서 동시에 재생하는것에 의해 플레이어가 종료되어 재시작 해야했던 현상 수정
  6. 플레이리스트 종료시 상태바 아이콘이 아직 재생중이던 현상 수정
  7. 플레이어 폰트를 맥의 기본폰트인 “애플 산돌고딕”으로 수정
